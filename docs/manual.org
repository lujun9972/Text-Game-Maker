* 简介

Text-Game-Maker 是基于 Emacs Lisp 的文字冒险游戏制作框架。通过单个 Org 文件定义全部游戏世界。

**v2 架构由三个核心机制组成：**

- **Registry 全局注册表**（=tg-registry.el=）— 所有游戏实体（房间、物品、生物等）注册到全局哈希表中，通过 symbol 查找。零依赖。
- **Org 配置文件** — 单个 =game.org= 定义 6 类游戏实体（Rooms、Objects、Creatures、Dialogs、Shops、Quests），由 =tg-config-load= 解析并注册到 Registry。
- **Handler Chain Dispatch** — 玩家命令经过 9 步处理链：error → room-before → io → do → action → after → NPC → buffs-tick → turn-increment。

* 安装与快速开始

**依赖：** Emacs 27+（内置 org-element、cl-lib）

**安装：**

#+begin_src elisp
(add-to-list 'load-path "~/path/to/Text-Game-Maker")
(require 'tg)
#+end_src

**启动游戏：**

#+begin_src elisp
M-x tg-start RET path/to/game.org RET
#+end_src

**运行示例游戏：**

#+begin_src sh
bash sample/play.sh
#+end_src

或手动：

#+begin_src elisp
(load "sample/sample-game.el")
M-x play-sample-game
#+end_src

* 核心概念

** tg-room

#+begin_src elisp
(cl-defstruct tg-room
  symbol        ; 符号，唯一标识（如 'entrance）
  name          ; 显示名称（如 "地牢入口"）
  desc          ; 完整描述（首次进入时显示）
  short-desc    ; 简短描述（重复进入时显示）
  exits         ; 出口 alist，如 ((north . cell) (east . hall))
  contents      ; 房间内物品 symbol 列表
  creatures     ; 房间内生物 symbol 列表
  before-handler ; 命令前回调函数
  after-handler  ; 命令后回调函数
  visit-count)   ; 访问次数
#+end_src

** tg-object

#+begin_src elisp
(cl-defstruct tg-object
  symbol    ; 唯一标识（如 'iron-sword）
  name      ; 显示名称（如 "铁剑"）
  synonyms  ; 同义词列表，用于命令匹配
  contents  ; 容器内物品（容器对象专用）
  supports  ; 支撑物上的物品（支撑物对象专用）
  props     ; 属性列表：wearable, edible, readable, container, supporter, openable, lockable, scenery, static
  state     ; 容器状态：nil, open, closed, locked
  key       ; 解锁所需物品 symbol
  effects   ; 效果 alist，如 ((attack 5) (defense 3))
  handler)  ; 自定义回调函数
#+end_src

** tg-creature

#+begin_src elisp
(cl-defstruct tg-creature
  symbol         ; 唯一标识
  name           ; 显示名称
  attr           ; 属性 alist，如 ((hp 100) (attack 5) (defense 3))
  inventory      ; 背包物品 symbol 列表
  equipment      ; 已装备物品 symbol 列表
  exp-reward     ; 被击败时给予玩家的经验值
  behaviors      ; NPC 行为规则列表
  death-trigger  ; 死亡触发回调
  shopkeeper     ; 是否为商人（t/nil）
  handler)       ; 自定义回调函数
#+end_src

=attr= 通过 =tg-creature-attr-get= 读取（alist 查询），=tg-creature-take-effect= 增量修改。=equipment= 中的对象 effects 在 =tg-creature-effective-attr= 中动态累加，计算公式：=base + equipment-bonus + buff-bonus=。

** tg-game

=tg-game= 不是 struct，而是哈希表。通过 =tg-game-get= / =tg-game-put= 访问。

常用键：

| 键 | 类型 | 说明 |
|---+---+---|
| =:title= | string | 游戏标题 |
| =:author= | string | 作者 |
| =:location= | symbol | 当前房间 |
| =:player= | symbol | 玩家生物 symbol |
| =:state= | symbol | 游戏状态（playing/paused/ended） |
| =:turns= | integer | 回合数 |
| =:active-buffs= | list | 当前生效的 buff 列表 |

** tg-dialog-state / tg-dialog-option

#+begin_src elisp
(cl-defstruct tg-dialog-state
  node-id     ; 当前对话节点 ID
  npc-symbol  ; NPC symbol
  greeting    ; 开场白
  options)    ; tg-dialog-option 列表

(cl-defstruct tg-dialog-option
  text       ; 选项文本
  response   ; NPC 回复
  condition  ; 显示条件（当前未实现，始终 nil）
  effects    ; 效果列表，如 ((exp 10))
  next-node) ; 跳转的下一个节点 ID
#+end_src

* Org 配置格式

配置文件为标准 Org 文件，包含头部关键字和 6 个 section。

** 头部关键字

| 关键字 | 必填 | 说明 | 示例 |
|---+---+---+---|
| =#+TITLE= | 是 | 游戏标题 | =#+TITLE: 地牢冒险= |
| =#+AUTHOR= | 否 | 作者 | =#+AUTHOR: DarkSun= |
| =#+START= | 是 | 起始房间 symbol | =#+START: entrance= |
| =#+PLAYER= | 是 | 玩家生物 symbol | =#+PLAYER: hero= |

** Rooms Section

每个房间为一个二级标题，属性通过 PROPERTIES drawer 定义。

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---+---+---+---+---|
| =NAME= | string | 是 | - | 房间显示名称 |
| =DESC= | string | 是 | - | 完整描述 |
| =SHORT_DESC= | string | 否 | 同 DESC | 简短描述 |
| =EXITS= | string | 否 | 无 | 方向=房间对，逗号分隔 |
| =CONTENTS= | string | 否 | 无 | 物品 symbol 列表，逗号分隔 |
| =CREATURES= | string | 否 | 无 | 生物 symbol 列表，逗号分隔 |

#+begin_src org
  ,** entrance
  :PROPERTIES:
  :NAME: 地牢入口
  :DESC: 沉重的石门在你身后缓缓关闭。
  :SHORT_DESC: 地牢入口
  :EXITS: north=cell,east=hall
  :CONTENTS: torch,rusty-key
  :CREATURES: guard
  :END:
#+end_src

** Objects Section

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---+---+---+---+---|
| =NAME= | string | 是 | - | 显示名称 |
| =SYNONYMS= | string | 否 | 无 | 同义词列表，逗号分隔 |
| =PROPS= | string | 否 | 无 | 属性：=edible=, =wearable=, =readable=, =container=, =supporter=, =openable=, =lockable=, =scenery=, =static= |
| =STATE= | symbol | 否 | nil | 容器状态：=open=, =closed=, =locked= |
| =KEY= | symbol | 否 | nil | 解锁所需物品 symbol |
| =EFFECTS= | string | 否 | 无 | 效果 alist，如 =(hp 30)= 或 =(attack 5) (defense 3)= |
| =CONTENTS= | string | 否 | 无 | 容器内初始物品，逗号分隔 |
| =SUPPORTS= | string | 否 | 无 | 支撑物上初始物品，逗号分隔 |
| =HANDLER= | string | 否 | nil | 自定义回调函数名 |

#+begin_src org
,** iron-sword
:PROPERTIES:
:NAME: 铁剑
:SYNONYMS: iron-sword
:PROPS: wearable
:EFFECTS: (attack 5)
:END:
#+end_src

** Creatures Section

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|---+---+---+---+---|
| =NAME= | string | 是 | - | 显示名称 |
| =ATTR= | string | 是 | - | 属性键值对，空格分隔 |
| =INVENTORY= | string | 否 | 无 | 初始背包物品 |
| =EQUIPMENT= | string | 否 | 无 | 初始装备 |
| =EXP_REWARD= | integer | 否 | 0 | 被击败时给予经验 |
| =BEHAVIORS= | string | 否 | 无 | 行为规则列表 |
| =DEATH_TRIGGER= | string | 否 | nil | 死亡触发回调 |
| =SHOPKEEPER= | t/nil | 否 | nil | 是否为商人 |
| =HANDLER= | string | 否 | nil | 自定义回调 |

=ATTR= 格式为键值对交替：=hp 100 attack 5 defense 3=。

=BEHAVIORS= 格式：规则列表，每条规则为 =(condition action args...)=。条件支持 =always=, =(player-in-room)=, =(hp-below N)=。动作支持 =attack=, =move random=, =say "text"=, =buff attr N=, =debuff attr N= 。

#+begin_src org
,** goblin
:PROPERTIES:
:NAME: 狡猾的哥布林
:ATTR: hp 25 attack 6 defense 2
:EXP_REWARD: 18
:BEHAVIORS: (((hp-below 13) say "你休想活着离开！") (always attack))
:SHOPKEEPER: nil
:END:
#+end_src

** Dialogs Section

每个对话为一个二级标题。PROPERTIES drawer 定义 NPC 和开场白，正文中每行定义一个对话选项。

| 字段 | 类型 | 必填 | 说明 |
|---+---+---+---|
| =NPC_SYMBOL= | symbol | 是 | 关联的 NPC symbol |
| =GREETING= | string | 否 | 对话开场白 |

选项格式：=[条件] 问题 :: 回复=，可选追加 =→ (奖励)=。条件方括号可选，支持 =has-item=, =quest-active=, =quest-completed= 及 =and=, =or=, =not= 组合。

#+begin_src org
,** prisoner
:PROPERTIES:
:NPC_SYMBOL: prisoner
:GREETING: 请救救我...你能帮帮我吗？
:END:
你是谁？ :: 我是被骷髅王关押的探险者。
有什么线索吗？ :: 武器库里有把好剑！ → (exp 15)
[(has-item bread)] 给你点面包 :: 谢谢你！ → (exp 10)
#+end_src

| =GOODS= | string | 是 | 商品=价格对，逗号分隔 |

** Quests Section

| 字段 | 类型 | 必填 | 说明 |
|---+---+---+---|
| =TYPE= | symbol | 是 | 任务类型：=kill=, =collect=, =explore=, =talk= |
| =TARGET= | symbol | 是 | 目标实体 symbol |
| =COUNT= | integer | 否 | 目标数量（默认 1） |
| =DESCRIPTION= | string | 否 | 任务描述文本 |
| =COMPLETION= | string | 否 | 任务完成时显示文本 |
| =REWARDS= | string | 否 | 奖励列表，如 =(exp 50) (item map)= |

:PROPERTIES:
:TYPE: kill
:TARGET: rat
:COUNT: 1
:DESCRIPTION: 消灭地牢里的老鼠
:COMPLETION: 你消灭了那只老鼠！
:REWARDS: (exp 10)
:END:

** Level Section

| 字段 | 类型 | 必填 | 说明 |
|---+---+---+---|
| =EXP_TABLE= | 逗号分隔整数 | 否 | 升级经验表（覆盖默认值） |
| =BONUS_POINTS= | integer | 否 | 每级自由属性点（覆盖默认值） |
| =AUTO_UPGRADE= | 属性键值对 | 否 | 每级自动提升属性（覆盖默认值） |

#+begin_src org
,* Level
:PROPERTIES:
:EXP_TABLE: 0,50,120,220,350,500,700,950,1300,1700
:BONUS_POINTS: 3
:AUTO_UPGRADE: hp 10
:END:
#+end_src

三个字段全部可选。缺失时保留 =tg-level.el= 中的 defvar 默认值。

* 扩展开发

** 注册自定义命令

使用 =tg-register-action= 注册新动词：

#+begin_src elisp
(tg-register-action
 :id 'search
 :synonyms '("search" "investigate")
 :handler
 (lambda (ast game)
   (let ((do-key (plist-get ast :do-key)))
     (cond
      ((not do-key)
       (tg-message "你要搜索什么？"))
      (t
       (tg-message "你仔细搜索了%s..." (tg-object-name (tg-get-object do-key)))
       ;; 随机发现隐藏物品的逻辑
       t)))))
#+end_src

注册后玩家可以输入 =search torch= 或 =investigate torch=。需要在 =tg-register-builtins= 之后调用。

** handlers.el 回调

在游戏配置目录的 =handlers.el= 中可定义各种回调：

- **房间 before-handler** — 命令执行前调用，返回 =t= 阻止后续处理
- **房间 after-handler** — 命令执行后调用
- **对象 handler** — 物品被操作时调用
- **生物 death-trigger** — 生物死亡时调用
- **生物 handler** — 生物被交互时调用

** 输出拦截

通过 =tg-message-hook= 拦截游戏输出：

#+begin_src elisp
(add-hook 'tg-message-hook
          (lambda (text)
            ;; 记录游戏日志
            (with-current-buffer "*game-log*"
              (insert text))))
#+end_src

** 内置命令注册流程

=tg-register-builtins= 注册 27 个内置命令。它由 =tg-init= 自动调用。自定义命令应在 =tg-init= 之后注册。
